Updating docker containers with Portainer on linux

January 15, 2023 2-minute read

How to update docker containers & portainer on your homelab.

I got into the whole Portainer craze when I found out about containers and their uses. This really cool tech lets you manage your servers utilizing containers. In short….containers are lightweight, standalone applications that let you quickly build, test and deploy applications. It also helps you manage complex server setups when you want to run different software without worrying about them conflicting with each other. They are isolated from one another so from a management standpoint , its much easier to manage and less stressful when dealing with server updates & upgrades.

Portainer is a web based GUI software you can install that will help you manage all of these docker containers. This blog post simply outlines the key elements with updating the Portainer software as well as the docker containers.

I followed novaspirits Pi-hosted series on youtube for the walk through: Pi-hosted’s update portainer video

Using a shell script to update portainer

Instead of typing out a couple commands in the terminal to properly update portainer, I decided to use this great resource from the Pi-hosted github repo. So 1st we will clone that repo to make use of the script he developed:

In the directory of your choosing, grab the code from novaspirits github: git clone https://github.com/novaspirit/pi-hosted

Then navigate into the pi-hosted directory to see the script we will run. Appropriately named update_portainer.sh: cd pi-hosted

List the files in the current directory to see the update shell script: ls

Before we run the update script, we will first update & upgrade our server itself. I do that with:

sudo apt update && sudo apt upgrade

Once that finishes, we will pull the latest code from the github repo to make sure we have the latest updates before we run the script: git pull then sh ./update_portainer.sh

Portainer will now be updated.

Update the containers inside portainer

  • Hop into portainer and find the list of containers in the container module.
  • Click the container you want to update
  • Stop the process by clicking the STOP button.
  • Recreate continer by clicking the RECREATE button. Enable “pull latest image” then click recreate.
  • Lastly, click START on the container you just recreated to get it running.

All of the steps above will have updated your server, updated your Portainer software and updated your containers within portainer.

Enjoy!

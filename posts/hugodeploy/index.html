Deploying Hugo public site via github hosting

define Hugo:

Hugo is a fast and modern static site generator written in Go, and designed to make website creation fun again.

At this point you should have 2 repos for your hugo website uploaded to github.

  1. source code repo
  2. public deployment repo

make directories for both repos locally.

mkdir sourceCodeHugoSite
mkdir publicDeploymentHugoSite

Then clone them into their respective directories.

Note; You can use submodules here but I found it much easier and less error prone to just create 2 separate directories to manage the public deployment

sourceCodeDir: git clone linkToSourceCodeRepoOnGithub
publicSiteDir: git clone linkToPublicRepoOnGithub

ALWAYS ALWAYS ALWAYS git pull before you start working on either project. ( source code or public ). Failing to do so will break your deployment.

sourceCodeDir: git pull
publicSiteDir: git pull

At this step you can update your source code website and add content. ( inside source code repo) Once finished you need to generate the public site code for deployment.

hugo -d ../directoryOfPublicSite

The cmd above will compile the files for public deployment and add to the other public site directory locally.

Now we are ready to save our changes:

git status
git add .
git commit -m "comments here"
git push

Should not have any errors in this repo.

Now change directory to the public site directory:

cd ../publicSiteDir

git status
git add .
git commit -m "comments here"
git push

Now navigate to your github.io page and the changes should be updated shortly. Sometimes you need to clear cookies / cache to see the change.

How to fix git issue with deploying hugo public site

September 14, 2021 One-minute read

I kept breaking my site deployment because of an issue with handling the multiple repos and folders from multiple computers.

To summarize:

When making a hugo deployment site, you should have 2 separate folders. 1 for the source code and the other for public deployment.

  1. Clone both repos to their own folders locally.
  2. git pull inside both so you are up to date with the remote repo.
  3. Once changes are made in source code, push the changes.
  4. hugo -d path/to/public/deployment
  5. CD into public deployment and git add, commit , push.

I will add a better summary once all errors are worked through and commited.

Using filebrowser in portainer to access fileserver

April 18, 2023 2-minute read
h4ckery.b1nx
portainer
portainer • filebrowser • fileserver

I run a Samba server on my home server to backup files, host pictures, movies, and more. Recently, I discovered a new feature called “filebrowser” that adds a web interface to access files on my server in a web browser of my choice. With filebrowser, I can simply browse to the URL of the filebrowser address, log in with my username, and then have access to whatever files the admin allows.

Why use filebrowser?

Using filebrowser has several advantages. My Samba server serves files that need to be accessed by multiple users. While this was all set up with the Samba configs, accessing the files required users to open up a folder on their desktop and hope the stickied network location to the Samba server was there. They then had to log in and access everything from there. Additionally, setting up all the folders and users inside config files made things a little complicated for users who weren’t tech-savvy or who only accessed the server every 4-6 months. They often forgot where to look or how to access their files.

Filebrowser solves these problems. Once I set up filebrowser on my server and bookmarked the address in users’ web browsers, all they have to remember is to click the bookmark to the filebrowser, log in, and then voila! They can see all their folders right there in the web browser. They can upload, download, move files around, and more.

In my opinion, filebrowser is a must-have for anyone hosting a file server on their home network. Its ease of access makes it worth it.
